## 목적

이 문서는 `{{PROJECT_NAME}}`에서 작업하는 AI 코딩 에이전트를 위한 공통 작업 기준입니다.

## 공통 원칙

- 변경은 작고 검증 가능하게 유지합니다.
- 기존 구조와 패턴을 먼저 따르고, 새 추상화는 필요할 때만 도입합니다.
- 동작 변경이 있으면 사용자 흐름 기준으로 영향 범위를 설명합니다.
- 비밀값, 토큰, 운영 데이터를 문서나 로그에 남기지 않습니다.

## 기본 작업 절차

1. 수정 대상과 인접한 파일을 먼저 읽습니다.
2. 서버/클라이언트 경계, 데이터 흐름, 테스트 범위를 확인합니다.
3. 최소 범위로 수정합니다.
4. 가능한 가장 작은 검증부터 실행합니다.
5. 최종 보고에는 변경 파일, 변경 이유, 실행한 검증, 실행하지 못한 검증을 포함합니다.
